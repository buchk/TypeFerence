# Branch report: feature/overnight-improvement-pass

Written for a skeptical senior reviewer. Every claim below was verified by running
the thing, not by reading the code; commands and outcomes are reproducible from
this branch. Nothing was pushed, tagged, or published.

## Completed, per priority

**Priority 1 — Go implementation + conformance suite: complete.**
Independent Go implementation under `go/` (module `github.com/buchk/TypeFerence/go`):
loader with strict YAML decoding, the full structural type system (embedding,
depth-based promotion, same-depth ambiguity errors, structural interface
satisfaction, capability contract preservation), deterministic compilation to all
four targets, trust configuration/signature handling, ARD catalog emission, and a
CLI (`validate`, `build`, `inspect`, `diff`, `eval`, `version`) that builds as a
single static binary (`CGO_ENABLED=0`, ~6 MB, cross-compiles to
linux/darwin/windows × amd64/arm64 — all six verified locally). Byte-compatibility
rests on `go/internal/jsonx`, a purpose-built JSON writer/parser whose escape table
was derived by **empirically probing System.Text.Json on .NET 10**, not recalled.
The Go binary reproduces the committed C#-built `dist/` byte-for-byte, including
the ARD catalog (`typeference diff examples/helio --against dist …` exits 0).

**Cross-implementation conformance suite: complete.** 26 language-neutral fixtures
under `conformance/` (20 success, 6 error), covering composition edges, five-level
embedding chains, shallowest-wins and ambiguity semantics, capability
promotion/override, structural interfaces, unicode + the full escape set,
CRLF/BOM/missing-trailing-newline byte edges, number-token preservation,
promoted-skill context isolation, digest path ordering, trust manifests (unsigned,
fail-closed, externally signed), and ASCII key-space enforcement. Digests are
generated by the Go runner and independently verified by the C# runner; **current
status: 26/26 pass on both implementations.** Both runners execute in CI.

**Priority 2 — self-hosting: complete.** `agents/maintainer/` defines this
repository's maintainer in TypeFerence (four profiles, two real capabilities);
root `AGENTS.md` and `dist-maintainer/` are its compiled artifacts with ARD-carried
provenance to the source digest; the CI `selfhost-drift` job fails on any drift
(demonstrated to trip on a mutated context file, then restored to zero drift).
Five type-system limitations found during self-hosting are recorded as design
feedback in ADR-0006 — arguably the most valuable output of that exercise.

**Priority 3 — eval harness: complete (dry-run verified; live untested).**
`typeference eval` runs scenario+rubric files against a resolved definition with
an LLM judge. Dry run is the default and CI mode: exact Messages API payloads
emitted, zero network. Live mode is implemented (stdlib HTTPS, `claude-opus-4-8`,
adaptive thinking, structured-output judge) and covered by an injected-backend
test, but **was never exercised against the real API** — no key was available in
this environment. Three starter scenarios target the helio agents. Framing in
`evals/README.md` is deliberately narrow: adherence signal, not equivalence.

**Release engineering (user-requested, done before P4/P5):** version 0.0.3 in
both CLIs (`typeference version`), tag-triggered `release.yml` (full test +
conformance + drift gate, six-platform archives + `SHA256SUMS`, GitHub Release via
`gh`, `v0.*` marked pre-release), `CHANGELOG.md`, `docs/release-checklist.md`,
ADR-0007 (no MSI — deliberate). **The workflow has never run**; it fires on the
first `v*` tag push after merge.

**Priority 4 — distribution: complete.** README quick start requires only Go:
clone → build → four commands, ending with a byte-identity proof against `dist/`.
Executed literally from a clean clone: **11.6 seconds** (target was two minutes).
The old `$tf` PowerShell alias is gone; .NET moved to a reference-implementation
section.

**Priority 5 — hardening: partial (see gaps).** Targeted C# tests for the
canonicalization rulings (code-point comparer, separator-independent digest against
a hand-computed SHA-256, ASCII slot names, `.yaml` pin); conformance fixture 026
pins digest path ordering cross-implementation; ARD integration explicitly pinned
to the v0.9 draft in the spec and alignment notes. Final counts: 73 C# tests,
full Go suite + 26-fixture conformance, gofmt/vet clean.

## ADRs written (docs/decisions/)

- 0001 — Record architecture decisions.
- 0002 — CLI verb is `typeference` (the prompt's `tf` premise was false; `$tf` was a README variable).
- 0003 — Go layout; stdlib-first with `gopkg.in/yaml.v3` as sole dependency; why `jsonx` exists.
- 0004 — Canonicalization rulings: per-artifact member order (spec's "ordinal key order" was wrong), normative escape table, raw number tokens, **digest path ordering was platform-dependent — fixed spec + C#**, code-point ordering with ASCII key spaces, ASCII trust identities, UTF-8/BOM/CRLF rules.
- 0005 — Conformance suite design: digest contract, generate-with-one/verify-with-other, error outcomes not stderr text.
- 0006 — Self-hosting + five type-system limitations (source-root sandbox vs governed files; no conditional norms; no invariant type; **context content invisible to neutral-target drift without ARD emission**; no version↔commit tie).
- 0007 — Release distribution: static archives + checksums, no installer, tag-driven gated pipeline.
- 0008 — Eval harness scope: Go-only, dry-run-first, stdlib HTTP over the SDK (with revisit criteria), judge design, what a pass does not establish.

## Review with extra care before merge

- **`docs/specification.md` changes** (Deterministic compilation, Artifact digest
  algorithm, ARD pin). These are normative. The key judgment call: canonical
  ordering defined as code point order with ASCII-restricted key spaces, rather
  than codifying .NET's UTF-16 ordinal behavior (ADR-0004 rulings 4–6).
- **Breaking validations**: non-ASCII slot names, trust metadata keys, and trust
  identities are now rejected by both implementations without a `schemaVersion`
  bump (justified by experimental-draft status; called out in the changelog).
- **C# `HashDirectory`/`PackageFiles` ordering change** — behavior-preserving for
  all committed corpora (golden hash test proves it) but a real change for trees
  with names straddling `0x2F`–`0x5C`; that's the platform bug being fixed.
- **`release.yml`** — has `contents: write`; publishes on any `v*` tag once merged.
  Review the gate steps and the `--prerelease` heuristic before the first tag.
- **`dist/` split** — `dist/` is now strictly the helio reference output;
  self-host artifacts live in `dist-maintainer/`. Anything downstream assuming the
  old `dist/maintainer` path breaks (all in-repo references were updated).
- **Environment residue** — Go 1.26.5 installed user-scope at
  `%LOCALAPPDATA%\go-toolchain`; a stale interrupted-rebase state on `main` was
  cleared with `git rebase --quit` (tree was clean, `main` == `origin/main`).
